# CODEX.md

## Purpose

Codex uses `AGENTS.md` as its generated runtime instruction file. Visual Studio-family agent surfaces may also use `AGENTS.md` as their homologated mirror. This file is a short adapter guide for humans and agents that need Codex-specific entrypoints.

## First Use

1. Confirm repo identity with `git remote -v` or project signals.
2. Run `python3 scripts/diagnose-first-use.py --dry-run`.
3. Run `python3 scripts/diagnose-user-context.py --dry-run` when durable user context may affect the task.
4. Run `python3 scripts/diagnose-personal-skills.py --dry-run` before relying on personal skills.
5. If the user only says `hola`, `buenas`, `hello`, `hey`, or `empecemos`, activate `/jm-adk:first-use`.
6. If the user gives an explicit task, collect only blocking context and proceed.
7. Keep `.jm-adk.local.json`, `.local/`, `.codex/`, `workspace/`, private `user-context/` content, and `.env*` out of git.

## Boundaries

- Codex and Visual Studio-family guidance is generated by `scripts/adapters/codex.sh` into `AGENTS.md`.
- Do not edit generated runtime docs without updating the adapter.
- Do not claim hooks, MCP, multimodal, or workspace automation unless verified in the current Codex environment.
- Treat `user-context/resources/` as private curated resources; open only indexed, task-relevant items.
- Treat `user-context/personal-skills/skills/` as the canonical private source for user-authored skills; never store them in root `skills/`.
- Keep `.local/skills/` as an ignored experiment or mirror cache, not durable source.

## Validation

```bash
python3 scripts/validate-onboarding.py
python3 scripts/check-devkit-readiness.py
python3 scripts/count-components.py --check-docs
```

## Placement & Naming Contract

`AGENTS.md` carries the runtime rules. In short: task deliverables → `workspace/{active}/artifacts/` (never mixed with system files); kit internals → maintainer mode only; filenames + slugs = kebab-case, concise, mnemonic. Full contract: `references/ontology/placement-naming-contract.md`.

Durable user context lives in `user-context/`, identified by
`user-context/.jm-adk-context.json`. Read `user-context/_INDICE.md` first and
write there only after explicit context-update instructions.

Create or improve personal skills with `python3 scripts/scaffold-skill.py --personal` and sync copy mirrors with `python3 scripts/sync-personal-skills.py --dry-run` before `--apply`.
