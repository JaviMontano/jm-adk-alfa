# Resources

Store curated, persistent user resources here only after explicit user approval.

Examples:

- CVs and professional profiles
- Identification documents
- Relevant URLs and source lists
- Private reference documents
- User-provided consultation material

This directory is private by default. Agents must read `_INDICE.md` or an explicit resource card before opening a resource, and must never bulk-load this directory.

Do not store passwords, tokens, API keys, private keys, or credentials here.
