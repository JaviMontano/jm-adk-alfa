# Funnel Analytics

Deterministic workflow for conversion funnel definition, tracking-plan audit, drop-off interpretation, and experiment-ready optimization planning.

## Triggers

- funnel-analytics
- funnel analytics
- conversion funnel
- drop-off analysis
- activation funnel
- checkout funnel
- tracking plan

## Allowed Tools

- Read
- Write
- Glob
- Grep
- Bash

## Quick Use

Use this skill when a user needs to define, inspect, or improve a digital conversion journey. Start by separating verified evidence from assumptions: event names, units, windows, denominators, and segment cuts must be explicit before rates or recommendations are treated as reliable.

## Minimum Inputs

- Business goal and funnel audience
- Candidate steps or journey description
- Event taxonomy, tracking plan, dashboard, data table, or code evidence
- Time window, timezone, and unit of analysis
- Known exclusions such as bots, employees, retries, refunds, or test accounts
- Privacy and data-governance constraints

## Output Format

Markdown report with scope, evidence inventory, funnel definition table, formulas, drop-off analysis, segment analysis, instrumentation gaps, optimization hypotheses, experiment backlog, validation plan, and risks.

## Not-Verified Rule

If a metric, event, source, denominator, or causal explanation is not supported by the provided artifacts, write `not verified` and list the next evidence needed. Do not invent analytics data.
