# UX Writing Checklist

Before delivering:

- Identify audience and source context.
- Build findings from source evidence before rewriting.
- Use evidence tags `[DOC]`, `[CONFIG]`, `[INFERENCIA]`, and `[SUPUESTO]`.
- Provide before/after rewrites for every requested label or message.
- Use Verb + Object for CTAs.
- For errors, include what happened, why when known, and how to fix.
- For empty states, include what is missing and the next action.
- Avoid color-only instructions and blaming language.
- Mark missing product facts as open; do not invent product claims.
