# Gratitud Post Proceso Body of Knowledge

## Stable Rules

- Good post-process gratitude is concrete: it names the person, the process, and one real contribution or topic.
- A thank-you message should reduce pressure, not create urgency or emotional debt.
- Differentiation is evidence-based: recruiter, interviewer, panelist, mentor, customer, and facilitator notes should not be interchangeable.
- Follow-up claims must be supplied by the user. Never imply acceptance, offer, commitment, timeline, or relationship warmth as fact.
- One distinctive brand phrase is the maximum per message; multiple slogans make the note feel performative.

## Message Shape

| Signal | Target |
|---|---|
| Recipient | Named person or clear persona |
| Evidence | One or more supplied interaction details |
| Tone | Warm, concise, confident, not servile |
| Next step | Present only when provided |
| Boundary | No FOMO, hustle, or pressure |

## Risk Signals

| Signal | Required Response |
|---|---|
| No recipient | Ask for recipient or provide blocked checklist |
| No process evidence | Draft only a generic-safe template and mark gap |
| FOMO phrase | Remove urgency and make follow-up neutral |
| Servile apology | Replace with concise appreciation |
| Outcome claim | Remove unless user supplied the fact |
