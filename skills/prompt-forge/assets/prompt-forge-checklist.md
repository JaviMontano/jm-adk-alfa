# Prompt Forge Checklist

Use this checklist before delivery.

- Mode is one of create, review, evolve, repair, or port.
- Target platform is explicit or marked `unknown`.
- Source boundary lists allowed sources and unsupported-source behavior.
- Playbook sections are present or omission is documented for batch/API use.
- Constraints include no invention and hidden-reasoning privacy.
- Output contract is present and preserved during evolve/repair.
- Rubric includes Foundation, Accuracy, Quality, Density, Simplicity, Clarity, Precision, Depth, Coherence, and Value.
- Scores below 8 include a repair.
- Tests include happy path, edge case, and adversarial cases.
- Porting includes source platform, target platform, mapped features, unsupported features, and losses.
- Current platform limits are sourced and dated or marked unknown.
- Remote assets are not required for rendering.
