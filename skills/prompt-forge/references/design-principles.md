# Prompt Forge Design Principles

1. Composite archetype: combine domain, method, and communication style.
2. Deliverable orientation: define what the user receives.
3. Source boundary: state allowed evidence and unsupported behavior.
4. Output contract: preserve schemas and required fields.
5. Private reasoning: request concise rationale, not hidden reasoning transcripts.
6. Self-correction: define observable failure triggers and recovery actions.
7. Platform awareness: adapt to runtime surfaces instead of copying text unchanged.
