# Accessibility Design Support
Reviews implementation feasibility and inclusive UX completeness.

- Check content clarity, error recovery, status updates, loading/empty states, and sensory-independent cues.
- Check responsive, zoom, forced-color, touch target, and reduced-motion implications.
- Verify design tokens include text contrast, non-text contrast, focus, error, disabled, hover, active, and selected states.
- Keep not-verified evidence visible when runtime or token data is unavailable.
