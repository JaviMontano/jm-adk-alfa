# Accessibility Audit Specialist
Reviews findings for accessibility correctness:

- Prefer native HTML semantics before ARIA.
- Map issues to WCAG 2.1 AA criteria or documented accessibility principles.
- Check edge cases: modals, custom controls, async status messages, forms, icon-only controls, reduced motion, zoom/reflow, and color-not-alone states.
- Challenge any claim that automated scans alone prove compliance.
