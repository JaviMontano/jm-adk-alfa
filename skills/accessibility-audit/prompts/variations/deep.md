# Accessibility Audit — Deep Analysis

Full depth execution for high-risk, public, regulated, or release-blocking UI.

1. Load `SKILL.md`, `knowledge/body-of-knowledge.md`, `templates/output.md`, and existing project accessibility tests.
2. Audit all in-scope routes/components and record not-verified areas explicitly.
3. Include automated scan evidence plus manual keyboard, focus, screen reader, contrast, forms/errors, dynamic content, motion, and responsive checks.
4. Map findings to WCAG 2.1 AA criteria or documented accessibility principles.
5. Produce owner-ready remediation tickets and block unsupported compliance claims.
