# Prompt Contract Checklist

Apply this checklist before returning or writing a prompt artifact.

## Source Gate

- Owning agent ID is provided by the user or discovered in a real source file.
- `sourceAgentMd` points to the inspected agent file or is marked `missing_source_agent`.
- Existing prompt paths were checked before choosing a filename.
- Prompt type is present in `assets/prompt-type-matrix.json`.
- Redirect types are not generated by this skill.

## Frontmatter Gate

- Required fields: `type`, `owningAgent`, `sourceAgentMd`, `version`, `createdBy`, `validationStatus`.
- `createdBy` is `prompt-creator`.
- `validationStatus` is `draft` until the artifact passes all gates.
- No invented tools, agents, commands, brands, dates, or quality gates.

## Body Gate

- Required sections: Purpose, Inputs, Procedure, Output Contract, Validation Gate, Failure Handling, Downstream Boundary.
- Type-specific sections from the matrix are present.
- Placeholders use descriptive snake_case: `{{task_id}}`, `{{source_agent_id}}`, `{{handoff_summary}}`.
- Avoid generic placeholders such as `{{x}}`, `{{var}}`, `{{thing}}`, or `{{value}}`.
- Do not include hidden reasoning, secrets, private user context, or unsupported external URLs.

## Decision Gate

- Return `write_prompt` when enough source context exists.
- Return `ask` when prompt type, owning agent, target agent, or success criteria are missing.
- Return `redirect` for `agent_system_prompt` and `workflow_step_prompt`.
- Return `decline` when the request asks for unsupported or unsafe behavior.
