# Mcp Engineering Deep Variation

Úsala cuando el scope es ambiguo, hay un secreto posiblemente filtrado, o el contrato de error debe cubrir muchas categorías.

Incluye: notas de descubrimiento (¿quién hereda el servidor?, ¿qué secretos existen en el historial?), opciones consideradas (`.mcp.json` vs `~/.claude.json`, MCP vs built-in), contrato de error completo con categorías y mapeo a códigos del transporte, plan de remediación de secretos (rotar + `git filter-repo`), política de reintento del cliente, validación offline si aplica y riesgos.
