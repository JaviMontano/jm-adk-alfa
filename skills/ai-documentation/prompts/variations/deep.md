# AI Documentation - Deep Mode

Use for multi-file documentation refreshes, API reference generation, runbooks,
or documentation drift audits.

Steps:

1. Inventory code, specs, configs, tests, and existing docs.
2. Compare existing docs against source evidence.
3. Produce target sections with evidence ids.
4. Separate blocking gaps from non-blocking gaps.
5. Validate with the offline script before delivery.
