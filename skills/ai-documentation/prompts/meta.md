# AI Documentation - Self-Improvement

Review the skill when:

- false-positive activation occurs for non-technical writing tasks
- a new documentation target type is needed
- output paths or evidence rules need stricter validation
- fixtures no longer cover a recurring failure mode

Improvement rules:

- add eval cases before changing behavior
- update assets before validator code when a closed policy changes
- keep validation offline and deterministic
- do not add external documentation services as required runtime dependencies
