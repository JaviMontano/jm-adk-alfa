# Tool Use Design Assets

These assets define deterministic tool-description routing contracts and offline validation.

## Inventory

- `manifest.json`: declares deterministic assets.
- `tool-use-contract.json`: required JSON report shape.
- `description-contract-policy.json`: required fields per tool description.
- `boundary-policy.json`: reciprocal boundary and delegation rules.
- `repo-strategy-policy.json`: `grep -> read -> edit` and read-all blockers.
- `edit-safety-policy.json`: unique-anchor failure and fallback.
- `anti-pattern-policy.json`: generic descriptions and overload blockers.
